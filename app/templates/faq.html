FAQ

What is LogSense AI?

A local tool for reading system logs. Upload a log file to get structured analysis, or connect an Android device to monitor live logcat output.

What types of logs can I upload?

Files with .txt, .log, or .xml extensions. The parser handles Windows, Linux, macOS, and Android log formats.

How do I get log files from my device?

Each OS has its own export method. The Instructions page covers the steps for each platform.

Do I need ADB?

Only for live Android monitoring. File upload works without ADB.

Which systems support ADB monitoring?

Linux, macOS, and Windows. ADB must be installed and accessible from the terminal. On Linux: sudo apt install adb. On macOS: brew install android-platform-tools. On Windows: download from the Android developer site.

My phone isn’t showing up. What should I check?

Check that USB debugging is on and the device is authorized. On Windows you may need OEM USB drivers. On Linux you may need udev rules. Try running adb devices in a terminal to check if the device is visible.

Can I filter Android logs?

Yes. The live terminal has level buttons (V/D/I/W/E/F) and a tag/keyword filter. Warning and Error are on by default.

Can I save live logs?

Yes. The Save button in the terminal exports all visible entries as a .txt file.

The live stream keeps dropping. Why?

Usually a cable or USB power issue. Try a different cable or port. You can also run adb devices in a terminal to check if the connection is stable.
