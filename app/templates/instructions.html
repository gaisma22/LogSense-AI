How to Use LogSense AI

Two modes: upload a log file for analysis, or connect an Android device for live monitoring.

Log Analysis

1. Upload a File

Go to Upload. Drop or select a .log, .txt, .xml, or .evtx file. Max 64MB. Click Analyze.

2. Read the Results

Results are grouped by section and sorted by severity. Use the severity dropdown to filter by Errors, Warnings, or Info. Use the threshold control to adjust how strictly entries are classified. Strict flags more, Relaxed flags less.

3. Search and Export

The search box filters against the raw log line and explanation text. Export filtered results as TXT, CSV, or JSON using the Export button. Raw downloads the original file unchanged.

Windows Event Logs

Open Event Viewer

Press Windows key + R, type eventvwr, press Enter. In the left panel, expand Windows Logs. You will find System, Application, Security, Setup, and Forwarded Events.

Which log to export

System: hardware issues, driver failures, service crashes.
Application: app errors and crashes from installed software.
Security: login attempts and access control events. Requires admin rights.

Option A: Export as .evtx

Right-click any log. Select Save All Events As. Set file type to Event Files (*.evtx). Save it anywhere. Upload the file on the Upload page.

Option B: Export as text

Same steps but set file type to Text (Tab-delimited) (*.txt). Upload the resulting .txt file.

Android Live Monitoring

1. Start Streaming

Click Check Device on the home page. Once the device shows as connected, click Start. The terminal opens as a floating overlay. Warning and Error entries show by default.

2. Read the Stream

Each row shows the timestamp, level, tag, and message. Hover any row to see a quick triage summary. Click a row to pause the stream and open the detail panel.

3. Detail Panel

The panel shows the triage verdict, what the entry means, which component logged it, and when it was recorded. The chart below shows log activity across the current session. Each bar represents a group of entries colored by severity. The marked bar is the entry you selected. Hover any bar to see what was happening at that point.

4. Controls

Pause freezes the stream without closing it. Resume picks up where it left off. Save exports all visible entries as a .txt file. Minimize collapses the overlay to a pill so you can use other pages while monitoring continues.

5. Filters

Use the level buttons (V D I W E F) to show or hide entries by level. Warning and Error are on by default. Use the tag or keyword filter to narrow down to a specific component.

6. Common Issues

  • USB debugging must be on in Developer Options.
  • Tap Allow on the phone when the authorization prompt appears.
  • On Windows, OEM USB drivers may be required for the device to show up.
  • On Linux, you may need udev rules. Run adb devices to check.
  • If the stream keeps dropping, try a different cable or USB port.

Tips

  • Files up to 64MB are supported. The parser caps at 20,000 lines to keep analysis fast.
  • Filter to High or Moderate before exporting to keep reports focused.
  • Use the threshold control to tune how aggressively entries are flagged.
  • Turn on I (Info) or D (Debug) in the terminal if Warning and Error alone are not enough.
  • The Pause button freezes the stream without closing it.
  • Minimize the terminal to keep monitoring while you use other pages.
