How to Use LogSense AI

Two modes: file upload for static logs, and live monitoring for connected Android devices.

Log Analysis

1. Upload a File

Go to Upload. Drop or select a .log, .txt, or .xml file. Click Analyze.

2. Read the Results

Results are grouped by section and severity. High severity entries show first by default.

3. Filter and Search

Use the severity filter to cut through noise. The search box matches against the raw log line and explanation.

4. Export

Export filtered results as TXT, CSV, or JSON. The Raw button downloads the original unmodified file.

Android Real-Time Monitoring

1. Connect Your Device

Enable USB debugging on the phone. Connect via USB. Click Check Device on the home page.

2. Start Streaming

Once the device shows as connected, click Start. The terminal opens as a floating overlay. Warning and Error entries show by default. Hover any row for a quick summary. Click a row to pause the stream and read the full detail in the side panel.

3. Common Issues

  • USB debugging must be on in Developer Options
  • Tap Allow on the authorization prompt on the phone
  • Run adb devices in a terminal to verify the connection
  • ADB must be installed: sudo apt install adb on Linux

Tips

  • Large files take longer. Files over 10,000 lines may take a few seconds.
  • Filter to High or Moderate before exporting to keep reports focused.
  • In the live terminal, hover a log row to see a quick explanation. Click to pause the stream and inspect the full entry.
  • Use the level buttons to hide noise. Warning and Error are on by default. Turn on Info or Debug if you need more detail.
  • The Save button in the terminal exports all visible entries as a .txt file.
  • Minimize the terminal to keep monitoring while you use other pages.
